## 1. Business Purpose

주식 투자에 관심은 있지만 실제 자금 손실이 두려워 시작하지 못하는 사람들이 많다.
특히 주식 투자 경험이 없는 입문자나 대학생들은 기본적인 매수/매도 개념조차 낯설어
실전 투자로 이어지지 못하는 경우가 대부분이다.

MockTrade는 가상 자금을 이용해 실제 주식 시장과 동일한 환경에서 매매를 경험할 수 있는
모의 주식거래 앱이다. 사용자는 금전적 리스크 없이 투자 전략을 세우고 실험해볼 수 있으며,
포트폴리오 관리와 수익률 분석을 통해 실전 투자 역량을 키울 수 있다.

**주요 목표:**
- 주식 투자 입문자가 부담 없이 매매를 경험할 수 있는 환경 제공
- 실시간 시세 기반의 현실감 있는 모의 거래 구현
- 포트폴리오 분석 및 수익률 추적으로 투자 학습 효과 극대화
- 랭킹 시스템을 통한 경쟁 요소 도입으로 지속적인 참여 유도

**타겟 시장:** 주식 투자 입문자, 재테크에 관심 있는 대학생 및 20~30대

## 4. Concept of Operation

### 1). Register

| | |
|---|---|
| Purpose | 신규 사용자가 앱을 이용할 수 있도록 계정을 생성 |
| Approach | 이메일, 비밀번호, 닉네임을 입력하고 가입 버튼을 누르면 서버에서 중복 이메일 여부를 확인 후 계정을 생성한다. 가입 완료 시 가상 자금 1,000만원이 자동 지급된다. |
| Dynamics | 앱을 처음 실행하거나 계정이 없는 경우 |
| Goals | 회원가입을 통해 로그인 및 모든 기능을 사용할 수 있도록 한다. |

### 2). Login

| | |
|---|---|
| Purpose | 등록된 사용자인지 인증 |
| Approach | 이메일과 비밀번호를 입력 후 로그인 버튼을 누르면 서버에서 회원 정보를 조회하여 성공/실패 여부를 반환한다. 실패 시 오류 메시지를 출력한다. |
| Dynamics | 앱 실행 시 또는 로그아웃 후 재접속할 경우 |

## 5. Problem Statement

MockTrade는 실시간 주식 시세 기반의 모의 거래 앱으로, 아래와 같은 기술적 문제와 제약사항을 고려해야 한다.

1. **실시간 시세 API 의존성:** 외부 주식 시세 API의 응답 속도 및 요금 정책에 따라 데이터 갱신 주기가 제한될 수 있다. API 장애 시 시세 조회가 불가능해질 수 있으므로 캐싱 전략이 필요하다.

2. **동시 접속 처리:** 다수의 사용자가 동시에 매수/매도 요청을 보낼 경우 서버 부하가 발생할 수 있다. 트랜잭션 처리와 동시성 제어가 필요하다.

3. **데이터 정합성:** 매수/매도 처리 중 네트워크 오류가 발생할 경우 가상 자금과 보유 종목 데이터 간 불일치가 생길 수 있다.

4. **모의 거래의 한계:** 실제 시장의 호가창, 슬리피지, 거래 수수료 등을 완벽히 재현하기 어렵기 때문에 실전 투자와 차이가 있을 수 있다.

5. **Non-Functional Requirements:**
   - 응답 시간: 시세 조회 및 거래 처리는 3초 이내 완료
   - 보안: 사용자 비밀번호는 암호화하여 저장
   - 가용성: 서비스 가동률 99% 이상 유지
   - 확장성: 사용자 수 증가에 따른 서버 수평 확장 가능 구조

## 6. Glossary

| Terms | Description |
|---|---|
| 가상 자금 | 회원가입 시 지급되는 실제 금전적 가치가 없는 모의 투자용 자금 (기본 1,000만원) |
| 매수 | 주식을 구매하는 행위. 가상 자금이 차감되고 보유 종목에 추가됨 |
| 매도 | 보유 주식을 판매하는 행위. 보유 수량이 차감되고 가상 자금이 증가함 |
| 포트폴리오 | 사용자가 현재 보유 중인 종목과 수익률 현황 |
| 수익률 | (현재가 - 평균 매수가) / 평균 매수가 × 100 (%) |
| 총 자산 | 가상 잔액 + 보유 종목 평가액의 합계 |
| 평균 매수가 | 동일 종목을 여러 번 매수했을 때의 평균 단가 |
| 랭킹 | 전체 사용자의 총 자산 기준 순위 |
| 시세 API | 실시간 주식 가격 데이터를 제공하는 외부 API |
| 관리자 | 회원 관리 및 서비스 운영 권한을 가진 특수 계정 |
