# EasyTrade 실행 및 배포 방법

## 1. 프로젝트 구성

EasyTrade는 Android 앱과 Spring Boot 백엔드로 구성되어 있습니다.
회원가입, 로그인, 주식 조회, 매수, 매도, 포트폴리오 기능은 백엔드 API 서버가 실행되어야 동작합니다.

```text
android/  - Android Kotlin 앱
backend/  - Spring Boot API 서버
```

## 2. 백엔드 서버 실행

로컬에서 백엔드를 실행할 때는 Java 21이 필요합니다.

```powershell
cd backend
.\gradlew.bat bootRun
```

배포용 JAR 파일을 만들 때는 다음 명령을 사용합니다.

```powershell
cd backend
.\gradlew.bat bootJar
```

빌드된 JAR 실행 명령은 다음과 같습니다.

```powershell
java -jar build/libs/easytrade-backend-0.0.1-SNAPSHOT.jar
```

## 3. KIS API 환경변수

실제 한국투자증권 KIS API를 사용하려면 백엔드 서버에 다음 환경변수를 설정해야 합니다.
실제 API 키와 시크릿은 GitHub에 업로드하지 않습니다.

```text
SPRING_PROFILES_ACTIVE=local
KIS_ENABLED=true
KIS_BASE_URL=https://openapi.koreainvestment.com:9443
KIS_APP_KEY=<한국투자증권 API 앱 키>
KIS_APP_SECRET=<한국투자증권 API 앱 시크릿>
JWT_SECRET=<JWT 서명용 비밀키>
```

## 4. Android 앱 API 주소 설정

백엔드 서버를 공개 주소로 배포한 뒤 Android 앱의 API 주소를 변경합니다.

수정 파일:

```text
android/app/src/main/java/com/easytrade/mobile/MainActivity.kt
```

예시:

```kotlin
private const val API_BASE_URL = "https://배포된-백엔드-주소/"
```

주소 끝에는 `/`를 포함해야 합니다.

## 5. APK 빌드

Android APK는 다음 명령으로 빌드합니다.

```powershell
cd android
.\gradlew.bat :app:assembleDebug
```

생성된 APK 위치:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

## 6. 시연 순서

권장 시연 흐름은 다음과 같습니다.

```text
회원가입 -> 로그인 -> 005930 검색 -> 종목 상세 화면 -> 선 차트 확인 -> 매수 -> 포트폴리오 확인
```

랭킹 및 전체 시장 화면은 여러 종목의 KIS API를 순차 호출하므로 네트워크 상태에 따라 느릴 수 있습니다.
